# 发布发行版

## 目标

把可以直接运行的便携压缩包发布到仓库的 `发行版 / Release` 页面，方便别人直接下载体验。

推荐上传的文件：

- `artifacts\LightStickyNote-win-x64-portable.zip`

不要把整个 `artifacts/` 目录提交进 Git 历史，发布附件更适合放到仓库的发行版页面。

## 日常发版步骤

### 1. 先生成便携包

直接双击项目根目录下的：

- `一键打包便携版.cmd`

打包完成后，会自动打开 `artifacts` 目录。正常情况下你会看到：

- `artifacts\LightStickyNote-win-x64-portable.zip`

### 2. 先把代码提交并推送

如果这一版有代码改动，先提交代码，再推送到远端：

```powershell
git add .
git commit -m "chore: prepare release"
git push origin main
```

如果你有不想提交的本地文件，比如 `打包命令.txt`，先不要把它加入暂存区。

### 3. 创建一个新的 Git 标签

发行版通常对应一个标签，建议使用下面两种命名方式之一：

- `v0.1.0`
- `v2026.05.31.1`

示例：

```powershell
git tag v2026.05.31.1
git push origin v2026.05.31.1
```

如果你已经有固定版本号习惯，就继续沿用自己的规则。

### 4. 到仓库网页创建发行版

打开你的仓库页面：

- [https://gitcode.com/qq_39776978/LightStickyNote](https://gitcode.com/qq_39776978/LightStickyNote)

然后按这个顺序操作：

1. 进入 `发行版 / Release` 页面
2. 选择刚刚推送的标签
3. 填写标题
4. 填写版本说明
5. 上传 `artifacts\LightStickyNote-win-x64-portable.zip`
6. 点击发布

如果 GitCode 页面入口位置以后有变化，也不用担心，核心流程始终是：

- 先推代码
- 再推标签
- 最后在发行版页面上传 ZIP 附件

## 推荐标题模板

```text
LightStickyNote v2026.05.31.1
```

## 推荐说明模板

```text
本次更新内容：

- 优化右侧边缘吸附隐藏体验
- 避免拖拽时被 Windows 分屏接管
- 优化自动收回逻辑，减少编辑时被打断
- 增加便携版一键打包入口

下载后解压 ZIP，直接运行 LightStickyNote.App.exe 即可使用。
```

## 下次最省事的做法

以后你每次发版，只需要记住这 4 步：

1. 双击 `一键打包便携版.cmd`
2. `git push origin main`
3. `git tag <版本号>` 然后 `git push origin <版本号>`
4. 去 GitCode 的发行版页面上传 ZIP

## 额外说明

- 便携包是自包含版本，用户下载后通常不需要额外安装 .NET。
- 便携版运行数据会保存在 EXE 同级目录下的 `user-data` 文件夹。
- 如果你替换新版本 EXE，最好保留原来的 `user-data` 文件夹，这样用户的数据不会丢。
